## 1. Final Extension Architecture

The extension compiles into a standard Chrome Manifest V3 extension bundle under `/dist` using Vite and CRXJS:

```
[Popup: Dojo Dashboard] <---> [chrome.storage.local] <---> [Background Service Worker]
  (src/App.tsx)                                              | (evaluates via Google Gemini API)
                                                             v
                                                  [Content Script: Overlay]
                                                    (src/content.tsx via DOM)
```

- **Popup (Dojo Dashboard)**: Saves the focus goal, toggles Focus Mode (Dojo Gate), manages the **Google Gemini API Key** securely, and renders daily statistics (Focus Stamina health bar, Evaluated pages, and Slashed distractions).
- **Background Worker**: Evaluates pages asynchronously against the user's active goal when tabs finish loading. It calls Google's **Gemini API** (`gemini-2.5-flash` model) with a custom system prompt and a strict `responseSchema` configuration. If a page is marked unproductive, it increments stats and dispatches a message to the content script. Handles tab close actions and whitelisting.
- **Content Script Overlay**: Injected into matched pages along with a compiled Tailwind stylesheet. Listens for warning messages, displaying a fullscreen Glassmorphic Dojo Card with Master Oji-San's sprite, speech bubbles, and action buttons.

## 4. Setup and Installation Instructions

Follow these steps to run and test the Oji-San Dojo Extension in Chrome:

### Step 1: Run the Production Build
Ensure all scripts compile and the `dist` folder is created by running:
```bash
npm run build
```

### Step 2: Load the Extension in Chrome
1. Open Google Chrome.
2. In the URL bar, go to `chrome://extensions/`.
3. In the top-right corner, toggle the **Developer mode** switch **ON**.
4. In the top-left corner, click **Load unpacked**.
5. Select the `/Users/rajkumar/Projects/personal/namastaeHackathon/dist` directory.

### Step 3: Configure and Spar
1. Click the **Extensions** icon (puzzle piece) in the Chrome toolbar and pin **Oji-San - AI Dojo Focus Master**.
2. Click the Oji-San icon to open the **Dojo Dashboard**.
3. Click the **Gear icon** in the top right to expand **Dojo Settings**.
4. Enter your **Google Gemini API Key** (starts with `AIzaSy...`).
5. Set your **Training Goal** (e.g. `Studying for AWS Certification` or `Learning React`).
6. Turn the **Dojo Gate** toggle **ON** (shows green "OPEN").
7. Navigate to a distracting page (e.g., a gaming video on YouTube or a social media page).
8. Verify that Master Oji-San intervenes, scolding you with dynamic text, and that clicking **Close Tab** closes the browser tab.

## 5. Bug Fixes (Completed: 2026-07-16)

- **UI Style Isolation**: Migrated the content script overlay in `content.tsx` to render inside a isolated **Shadow DOM** and loaded the Tailwind stylesheet dynamically inline via Vite `?inline` imports. This keeps Oji-San's card layout from being overridden by host page styles (like Netflix's CSS).
- **CSP Image Fix**: Converted `oji-san.png` to a **Base64 data URL** and stored it inside `oji-san-base64.ts`. This completely prevents Content Security Policy (CSP) image loading blockages on external sites.
- **Button Extension Context Handler**: Wrapped button clicks with validation checks that catch the `Extension context invalidated` error (which occurs when an extension is updated but the page has not been refreshed) and prompts the user to refresh their browser tab.
- **Invisible Toggle Button**: Corrected the slider knob width and height in `App.tsx` from non-existent `w-6.5 h-6.5` to standard `w-6 h-6` Tailwind CSS classes to make the Dojo Gate switch knob visible.
- **Lucide React Icons Integration**: Replaced all Material Symbols font icons with Lucide React SVG components in both the Popup Dashboard and the content script. This resolves Netflix's Content Security Policy (CSP) blocking external web fonts, which caused fallback words like `warning`, `gavel`, `history`, and `front_hand` to display as raw text and warp the layout.
- **YouTube Shield (Unhook Mode)**: Shifted YouTube Shield to an AI-free recommendations blocker to conserve Gemini API quota. When active, it injects high-specificity CSS rules to hide homepage video feeds (rendering a centered Dojo Search Banner instead), and hides the sidebar recommendations and comments completely on watch pages. This turns YouTube into a pure search utility with zero recommendation traps and zero API call consumption.
- **Persistent Evaluation Cache**: Implemented a URL/Domain evaluation cache in `background.ts` stored in `chrome.storage.local`. Once a URL is classified by Gemini or Groq, the result is cached persistently. Subsequent reloads or tab switches fetch directly from this cache, reducing API calls to exactly **one request per unique URL/Domain**.
  - **Domain-Level Block Caching**: Extended the cache to save unproductive classifications at the registered **Domain level** (e.g. `keka.com`, `netflix.com`) for all non-multi-purpose hosts. If any page on that domain is classified as off-topic, the entire domain is immediately blocked for the session without making any new API calls.
  - **In-flight Call Lock**: Implemented an `inFlightEvaluations` tracking registry that blocks duplicate, concurrent async requests for the same URL, resolving the double-evaluation race condition during page updates.
  - **Dojo Load Handshake**: Integrated a startup handshake (`checkInterventionStatus`) in `content.tsx` and `background.ts`. The content script queries the background immediately on startup to check if its active URL/domain is cached as unproductive. If yes, it blocks the page instantly, preventing page bypasses on fast redirects and resolving the "connection receiving end does not exist" message passing race.
  - **Connection Warning Suppressor**: Suppressed uncaught connection stack trace errors in `background.ts` by registering callbacks on `chrome.tabs.sendMessage` and capturing `chrome.runtime.lastError`.
- **SPA History Transitions**: Enabled background evaluation for single-page transitions (like YouTube's page changes) by listening to history URL changes in `onUpdated`.
- **Tab Switch Loophole Closer**: Registered a `chrome.tabs.onActivated` listener in `background.ts` that triggers an instant evaluation when a user switches tabs, preventing them from bypassing Oji-San by keeping unapproved tabs open.
- **YouTube Shield Toggle Switch**: Added a dedicated settings toggle card for the YouTube Shield in the popup dashboard, storing the state in local storage.
- **Groq API Integration**: Refactored `background.ts` and `App.tsx` to support **Groq** as an alternative, highly efficient, free-model AI provider. Disciple users can input their `groqApiKey` and choose specific Groq models (including `openai/gpt-oss-20b`, `llama-3.3-70b-versatile`, or `llama-3.1-8b-instant`). Evaluations run through Groq's OpenAI-compatible completions endpoint (`/chat/completions`) with strict JSON mode enabled (`response_format: { type: "json_object" }`).
- **Dojo Cache Clear Utility**: Added a "Clear Cache" button to the settings panel, allowing users to wipe the persistent URL evaluation cache and re-trigger fresh LLM evaluations for testing.
